## 第 2 頁

1-1 NodeMCU-32S 介紹
從國外傳來的創客(Maker)活動，在國內也同樣受到大眾的歡迎，並且快速
形成風潮，創客活動是透過動手做，將創意具體化成為作品，由於 Arduino 控制
板簡單易用，且價格便宜，所以 經常使用 Arduino 控制板來完成創意作品，而
Arduino UNO 是最常用的控制板，Arduino UNO 如圖 1-1 所示。
圖 1-1 Arduino UNO R3
近年來物聯網(Internet of Thing) 及人工智慧(Artificial Intelligence)的興起，創
客的創意開始有連接網路或是提高算力的需求，Arduino UNO 已經不敷使用，於
是市面上開始出現許多可以連接 WiFi 及藍牙(Bluetooth)的控制板，其中 ESP32 是
目前討論度很高的晶片組 ，ESP32 是樂鑫科技公司的產品 ， 其內建WiFi 及藍牙晶
片 ， 且相容於Arduino 的操作環境 ， 所以受到全球創客們的青睞 ， 不過使用 ESP32
晶片組的控制板種類眾多，本書採用安信可科技公司所生產的 NodeMCU-32S，
如圖 1-2 所示。
圖 1-2 NodeMCU-32S
NodeMCU-32S 採用 ESP32S 晶片組 ，具有 雙核處理器， 運算時脈速度比
Arduino UNO 快約 8 倍，算力大幅提升，且相容於 Arduino 的操作環境，可使用
Arduino IDE 進行開發，內建 WiFi 及藍牙的連線功能，WiFi 支援 2.4GHz，藍牙支
援傳統藍牙及低功耗藍牙(Bluetooth Low Energy)，另外還內建硬體時鐘(Real Time
Clock)，只要設定時間就可自動計時，電源部分提供有直流 5V 及 3.3V，可連接
5V 及 3.3V 的感測模組，價格的部分十分親民，在物聯網及人工智慧產業興起的
現今，NodeMCU-32S 補足 Arduino UNO 的缺口 ，同時搭配後面會提到的 EZ Start Kit，
讓我們輕鬆完成創意作品。

## 第 4 頁

1-2 EZ Start Kit 介紹及組裝
NodeMCU-32S 控制板基本配備齊全，不過控制板上只有內建 LED 及按鈕開
關，想要使用其他感測模組，必須要透過控制板上的排針接線，直接用排針接線
常會造成實作的問題，一般而言，我們會使用擴充板來解決接線不穩定，以及電
源針腳不足的問題，本書選擇 EZ Start Kit 作為擴充板，其內建多種感測模組，無
需接線，方便初學者專注學習程式撰寫，若用於課堂授課，可避免學生接線所造
成的錯誤，讓學習更佳順暢。
EZ Start Kit 為國內電子科技公司 iCShop 的產品，其可做為 NodeMCU-32S 的
擴充板之外，還可用於 micro:bit 及 LinkIt7697，內建感測模組有三顆 LED、三顆
WS2812 RGB LED、兩顆按鈕開關、可變電阻、光感測器、蜂鳴器、DHT11 溫濕度
感測器、紅外線接收器、繼電器以及 1.3 吋 OLED，總共 10 種感測模組，另外還
有擴充應用針腳及 ESP32 插槽可供外接感測模組， 所有模組及元件的配置位置如
圖 1-5 所示，本書在第二章會詳細說明內建感測模組及外接感測模組的使用。
圖 1-4 EZ Start Kit 感測模組位置說明
EZ Start Kit 的每個模組在電路板的正面都有印上 IO 編號，例如：紅色 LED
是 IO4，黃色 LED 是 IO5，綠色 LED 是 IO6，如圖 1-4 所示，這些 IO 編號的作用
是查詢所使用開發板上的 IO 腳位，因為 EZ Start Kit 可以接三種不同的開發板，
所以同一個 IO 編號接不同的開發板時，會對應到不同的開發板 IO 腳位，而 IO
編號對應開發板上的 IO 腳位表印在 EZ Start Kit 電路板的背面，如圖 1-5 所示，
以紅色 LED 為例，紅色 LED 的 IO 編號是 IO4，由對照表可知，IO4 對照到 ESP32
接腳是 GPIO16，對照到 7697 則是 P13，對照到 micro:bit 同樣是 P13，其餘以此
類推。撰寫程式前，我們必須查詢所使用模組的ESP32 腳位，如果程式中的腳位
填錯，會發生錯誤的動作。

後續會有詳細說明。

小。
8. 版本顯示：顯示 BlocklyDuino 版本，目前最新版本為 F2。
9. 積木程式與文字程式切換：積木型程式與所對應文字型程式的顯示切換。
10. 積木與 XML 程式切換：積木與所對應 XML 程式程式的顯示切換。
11. 積木顯示設定：設定程式編輯區的積木顯示，有放大、縮小等功能。
12. 垃圾桶：將程式積木拖曳至此區，即可刪除程式積木。
13. 檔案狀態：顯示目前存檔或開啟的檔案名稱。
